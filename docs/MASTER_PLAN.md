# Master Plan

## Project objective

Build a maintainable, restart-safe, data-persistent crypto trading bot through controlled paper/demo validation before any live-money consideration.

## Delivery sequence

1. Governance and repository foundation
2. Architecture lock
3. Market Data Engine
4. Strategy Engine
5. Risk Engine
6. Paper Execution Engine
7. Trade Management Engine
8. Persistent database and reconciliation
9. API and frontend dashboard
10. Monitoring, recovery, and security
11. End-to-end and soak testing
12. Controlled deployment

## Delivery rule

Only one bounded active module/task may be open at a time. Every module must complete implementation, tests, review, documentation, and verified handover before the next module begins.

## Current phase

Governance and repository foundation.
